# 🎧 Prompting 101 | Code w/ Claude

## 🧠 **ТЕЗИС**
Промпт-инжиниринг — это итеративная эмпирическая наука: правильная структура промпта, контекст и пошаговые инструкции превращают неуверенные ответы Claude в точные и применимые решения для реальных бизнес-задач.

## 🔑 **КЛЮЧЕВЫЕ ИДЕИ (7)**
• Промпт-инжиниринг = четкие инструкции + контекст + правильная структура информации
• Итеративный подход: тестируй → анализируй ошибки → улучшай промпт → повторяй
• Структура промпта: задача → контент → детальные инструкции → примеры → напоминание
• Claude любит структуру: XML-теги и markdown помогают организовать информацию
• Порядок анализа важен: сначала форма (факты), потом эскиз (интерпретация)
• Статическую информацию (форматы, правила) лучше помещать в system prompt
• Форматирование вывода через XML-теги или prefill упрощает интеграцию в приложения

## 🗺️ **СТРУКТУРА/СЕГМЕНТЫ**
• **Введение и проблема** — демонстрация неудачного промпта (лыжная авария вместо ДТП)
• **Структура хорошего промпта** — 10-шаговый подход к построению эффективного промпта
• **Контекст задачи и тон** — определение роли и требований к уверенности ответов
• **Фоновая информация** — добавление статических данных о форме в system prompt
• **Примеры и история диалогов** — использование few-shot примеров для сложных случаев
• **Детальные инструкции** — пошаговый алгоритм анализа с правильным порядком
• **Форматирование вывода** — структурирование результата для интеграции в приложения
• **Продвинутые техники** — prefill и использование расширенного мышления Claude

## ⚖️ **АРГУМЕНТЫ ↔ ВОЗРАЖЕНИЯ**
• **За структурированный подход**: Claude показал прогресс от "лыжной аварии" до точного определения виновника ДТП через итерации
• **За контекстную информацию**: без описания формы Claude тратил время на угадывание, с описанием — сразу переходил к анализу
• **За пошаговые инструкции**: порядок "сначала форма, потом эскиз" дал более точные результаты, чем хаотичный анализ
• **Потенциальная проблема**: слишком детальные инструкции могут привести к избыточному выводу (показ каждого чекбокса)

## 📊 **ДАННЫЕ/ЦИФРЫ**
• 17 чекбоксов — количество полей в шведской форме ДТП
• 10 шагов — рекомендуемая структура промпта от Anthropic
• Температура 0 — настройка для максимальной консистентности ответов
• 2 автомобиля (A и B) — структура анализируемой формы ДТП

## 📚 **ГЛОССАРИЙ (до 10)**
• **Промпт-инжиниринг** — практика написания четких инструкций для языковых моделей
• **System prompt** — постоянная часть промпта с контекстом и правилами, не меняющаяся между запросами
• **Few-shot** — предоставление примеров для обучения модели правильным ответам
• **XML-теги** — способ структурирования информации в промпте для лучшего понимания Claude
• **Temperature** — параметр, контролирующий случайность ответов (0 = максимальная предсказуемость)
• **Prefill** — техника "подсказки" начала ответа для контроля формата вывода
• **Prompt caching** — кеширование статических частей промпта для экономии токенов
• **Extended thinking** — режим расширенного мышления Claude с видимым процессом рассуждений
