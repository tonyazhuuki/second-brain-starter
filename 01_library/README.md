# 01_library — знания и источники

## Зачем
- База знаний: книги, подкасты/видео (TL;DR), практики, исследования.
- «Ядро смыслов»: сюда сходятся ссылки из целей/проектов, отсюда — в практики.

## MOC (Map of Content)
- Карты содержания хранятся в папке [`mocs/`](./mocs/). Создавай/поддерживай их по мере роста библиотеки.

## Формат
- Каждый файл имеет YAML‑фронтматтер (см. `.cursor/rules/yaml_conventions.mdc`).
- Ссылки — только относительные, с зеркальными бэклинками (см. `.cursor/rules/linking_policy.mdc`).

## 01_library — библиотека знаний

Рекомендуемая структура:
- `books/` — заметки по книгам
- `podcasts/` — заметки по подкастам (папки по темам)
- `practices/` — протоколы практик
- `research/` — исследования/концепты
