# Dreams Structurer — быстрый старт (Bundle A)

Мини‑пакет, чтобы разложить RAW‑мечты по сферам и получить читабельный `structured.md` + сводку Four Pillars. Работает «из коробки», без настроек и без интернета.

## Что внутри
- `tools/dreams_structurer.py` — скрипт (мульти‑разметка, Four Pillars, JSON отчёт)
- `00_vision/dreams/structured_template.md` — шаблон результата
- `00_vision/dreams/dreams_guidance.md` — гайд по процессу (RAW → Structured → семсрез)

Требования: Python 3.8+

## Как использовать (3 шага)
1) Скопируйте три файла/папки в свой репозиторий, сохранив структуру папок:
   - `tools/dreams_structurer.py`
   - `00_vision/dreams/structured_template.md`
   - `00_vision/dreams/dreams_guidance.md`

2) Подготовьте RAW‑мечты (свободный текст/буллеты):
   - Создайте файл с мечтами (например, `00_vision/dreams/raw.md`)
   - Пишите «как есть», без редактуры. Заголовки/цитаты/ссылки скрипт игнорирует

3) Запустите скрипт:
```bash
python3 tools/dreams_structurer.py --raw 00_vision/dreams/raw.md --out 00_vision/dreams/structured.md
```
Результат:
- `00_vision/dreams/structured.md` — мечты по сферам + блок «🧭 Four Pillars — покрытие мечт»
- `00_vision/dreams/structured.json` — машинно‑читаемый отчёт (для дашбордов)

Дальше (опционально): откройте `structured.md` и с помощью модели в редакторе добавьте в конец раздел «🧩 Выводы (семантический срез)» (сильные узлы, тонкие места, образы, мягкие усиления, вопросы).

## Важно знать
- Мульти‑разметка: один пункт может попасть сразу в несколько сфер (напр. «паспорт» → Документы + Пространство; «недвижимость» → Капитал + Пространство)
- Искусство (рис/арт/картины) не уезжает в «Здоровье», даже если встречаются слова «тело/секс» (это творчество)
- Документы влияют на образ жизни, поэтому дублируются в «Пространство»
- Приватность: скрипт не ходит в сеть; всё локально

## Частые вопросы
- «У меня другая структура папок — что делать?»
  - Передайте свои относительные пути в параметрах `--raw` и `--out` при запуске, например:
  ```bash
  python3 tools/dreams_structurer.py --raw notes/dreams_raw.md --out notes/dreams_structured.md
  ```
- «Можно кастомизировать названия сфер/ключевые слова?»
  - В Bundle A — нет (работает из коробки). Если нужна кастомизация под вашу терминологию, используйте Bundle B (с конфигом YAML).

## Мини‑диагностика результата
- Проверьте, не попали ли художественные формулировки про картины в «Здоровье» (должны быть в «Творчество»)
- Убедитесь, что «недвижимость» видна и в «Капитал», и в «Пространство»
- Если что‑то спорное — поправьте руками или повторно запустите с отредактированным RAW
