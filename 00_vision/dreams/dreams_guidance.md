# 🌌 Шаблон: Мечты (Dreams)

## Зачем (личная стратегия)
- **Главная цель**: из мечт собрать личную стратегию — то, **КАК** ты будешь их воплощать в главные цели и действия.
- Из стратегии кристаллизуются **цели** и **видение жизни**; мечты сохраняют энергию и направление.
- Краткий онбординг: [Личная стратегия — onboarding](../personal_strategy_onboarding.md)

## 🧪 Фреймворк: честные мечты (guidelines)
- **Честность и чистота**: формулируй без позы, без ожиданий «как надо». Своя интонация важнее красивости.
- **Анти‑грандиозность**: масштаб — не цель. Большие мечты допустимы, но не обязательны. «Маленькие и простые» мечты тоже валидны.
- **Связь с реальностью**: задача стратегии — соединять мечты с реальностью через:
  - рефрейминг (найти более близкую формулировку той же сути);
  - маленькие следующие шаги (safe‑to‑try за 15–60 минут);
  - отсечение факторов/энергоутечек, мешающих шагу;
  - сохранение «энергии мечты» (RAW‑заметки, частый короткий контакт).
- **Найти «правильные мечты»**: ориентир — телесный простор/энергия после чтения. Если мечта «красивая», но пустая или давящая — сузить до ближайшей правды.
- **Анти‑фрустрация**: если грандиозность вызывает разрыв и боль — уменьшить масштаб до действия на неделю.
- **Мини‑метрики**: 1) субъективная энергия (0–10) после формулирования ≥ 7; 2) есть один безопасный следующий шаг.

## Принцип RAW → Structured
1. Пиши в RAW «как чувствуешь» — свободный поток без редактуры (сохранить энергию и полноту смысла).
2. Затем структурируй в Structured по сферам и **связывай с целями года/месяца**:
   - Скриптом (есть в `tools/`): 
   ```bash
   python3 tools/dreams_structurer.py --raw 00_vision/dreams/raw.md --out 00_vision/dreams/structured.md
   ```
   Что делает скрипт:
   - раскладка по сферам с мульти‑разметкой (один пункт может жить в 2–3 сферах);
   - блок «🧭 Four Pillars — покрытие мечт» (faith/family/friends/service);
   - JSON рядом: `00_vision/dreams/structured.json` (для дашбордов).
   - Документы дублируются в «Пространство», недвижимость — в «Капитал» и «Пространство».
   - Искусство (рис/арт/картины) отделено от «Здоровья» даже если встречается «тело/секс».
   - При конфликте сфер используется тай‑брейкер, но мульти‑метки сохраняются.
   - Скрипт фильтрует заголовки/цитаты/ссылки.
   - Или промптом (если без скрипта):
   ```text
   Задача: разложить мечты из блока RAW по сферам (Здоровье, Отношения/Семья/Сообщество, Творчество, Работа/Служение, Пространство/Образ жизни, Капитал/Финансы). 
   Сохрани формулировки без украшательств, ничего не выдумывай. 
   Выведи markdown по шаблону structured_template.md с незаполненные места оставь как «—».
   ```

3. Семантический срез (LLM в редакторе — опционально):
   - Открой `00_vision/dreams/structured.md`.
   - Попроси модель: «Сделай краткий семантический срез мечт: сильные узлы, тонкие места, повторяющиеся образы, мягкие усиления без KPI, вопросы на размышление. Добавь раздел “🧩 Выводы (семантический срез)” в конец файла.»
   - Сохрани вывод прямо в конце файла под горизонтальной линией `---`.

4. Синхронизация с целями:
   - Еженедельно просматривай structured → перенос в `00_vision/goals/YYYY/MM.md` (только там, где уместно).
   - В месячнике используй блок «✅ 4 опоры — чек месяца» для мягкого балансирования.

5. Ручные оверрайды (по желанию):
   - В `raw.md` можно добавлять подсказки:
   `<!-- sphere: Творчество; Пространство --> 42. …`
   - Скрипт будет уважать ручные метки (если включить флаг в будущем).
